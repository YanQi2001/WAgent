AI/大模型面试八股知识要点

## Transformer 架构

Q: 请解释 Transformer 架构中的自注意力机制。
A: Transformer 的自注意力（Self-Attention）机制通过计算 Query、Key、Value 三个矩阵来建立序列中任意两个位置之间的直接关联。具体公式为 Attention(Q,K,V) = softmax(QK^T / sqrt(d_k)) * V。Multi-Head Attention 是将注意力分成多个头（head），每个头独立计算注意力，然后拼接起来，这样模型可以在不同的子空间中学习不同的注意力模式。

Q: 为什么 Transformer 要使用缩放点积注意力（Scaled Dot-Product Attention）？
A: 当维度 d_k 较大时，点积的值会很大，导致 softmax 函数进入梯度极小的区域（梯度消失），使得训练不稳定。除以 sqrt(d_k) 可以缓解这个问题，使得梯度更加稳定。

## RAG 检索增强生成

Q: 什么是 RAG（Retrieval-Augmented Generation）？它解决了 LLM 的哪些问题？
A: RAG 是一种将信息检索与大语言模型结合的技术范式。它解决了 LLM 的三个核心问题：（1）知识截止——LLM 的训练数据有时效性限制，RAG 可以接入最新数据；（2）幻觉问题——通过检索真实文档作为依据，降低模型胡编乱造的概率；（3）领域知识不足——通过接入特定领域的知识库，增强模型在垂直领域的表现。

Q: RAG 系统的典型流程是什么？
A: 典型的 RAG 流程包括：（1）文档预处理——切分、清洗、提取关键信息；（2）向量化——使用 Embedding 模型将文本转为向量；（3）索引——将向量存入向量数据库；（4）检索——根据用户查询检索最相关的文档片段；（5）增强生成——将检索到的上下文与用户问题拼接后送入 LLM 生成回答。

## Agent 架构

Q: 什么是 AI Agent？与传统的 LLM 应用有什么区别？
A: AI Agent 是能够自主规划、执行和反思的智能体。与传统 LLM 应用（单轮 prompt-response）的区别在于：（1）Agent 具有规划能力——可以将复杂任务分解为子步骤；（2）工具使用——可以调用外部 API、数据库、搜索引擎等；（3）记忆——可以保持上下文和长期记忆；（4）反思——可以评估自己的输出并进行自我纠正。

Q: 请解释 ReAct 框架的思想。
A: ReAct（Reasoning + Acting）框架让 LLM 交替进行推理（Reasoning）和行动（Acting）。具体来说，模型首先生成一个思考（Thought），然后根据思考决定采取什么行动（Action），执行行动后得到观察结果（Observation），再基于观察继续思考。这种交替模式使得模型能够更好地利用外部工具，同时保持推理链条的连贯性。

## 向量数据库

Q: 常见的向量数据库有哪些？各有什么特点？
A: 常见的向量数据库包括：（1）Qdrant——Rust 实现，支持丰富的过滤条件和 payload 存储，适合生产环境；（2）Milvus——C++ 核心，支持大规模分布式部署，适合十亿级向量；（3）Pinecone——全托管云服务，使用简单但成本较高；（4）Chroma——轻量级嵌入式数据库，适合原型开发；（5）FAISS——Meta 开源的向量检索库，不是完整数据库，但索引性能极强。

## Prompt Engineering

Q: 什么是 Chain-of-Thought（CoT）prompting？
A: Chain-of-Thought 是一种提示工程技术，通过在 prompt 中展示推理过程的中间步骤，引导 LLM 逐步推理而非直接给出答案。例如在数学问题中，展示"先计算A，再计算B，最后得到C"的过程。CoT 可以显著提升 LLM 在复杂推理任务上的表现，包括数学推理、多步逻辑推理、代码生成等。Zero-shot CoT 只需在 prompt 末尾添加"Let's think step by step"即可触发。
